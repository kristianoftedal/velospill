1. Install new dependencies:
   ```bash
   npm install pusher pusher-js @upstash/qstash use-sound
   npm install --save-dev @types/howler
   ```

2. Create `src/db/schema/draft.ts` with:

   - `draftStatusEnum`: pgEnum with values `['pending', 'men', 'women', 'complete', 'paused']`
     - `pending` = not yet started, `men` = men's draft active, `women` = women's draft active, `complete` = all picks done, `paused` = admin paused (future use)

   - `draftSessions` table:
     - `id`: serial PK (matching existing pattern)
     - `leagueId`: integer, NOT NULL, unique (one session per league), references `leagues.id` with `onDelete: 'cascade'`
     - `status`: draftStatusEnum, NOT NULL, default `'pending'`
     - `currentPickIndex`: integer, NOT NULL, default 0
     - `currentTeamId`: integer, nullable, references `teams.id`
     - `currentGender`: text typed as `$type<'M' | 'F'>()`, nullable
     - `timerExpiresAt`: timestamp with timezone, nullable
     - `startedAt`: timestamp with timezone, nullable
     - `completedAt`: timestamp with timezone, nullable
     - Index: `draft_sessions_league_idx` on `leagueId`

   - `draftPicks` table:
     - `id`: serial PK
     - `leagueId`: integer, NOT NULL, references `leagues.id` with `onDelete: 'cascade'`
     - `teamId`: integer, NOT NULL, references `teams.id`
     - `riderId`: integer, NOT NULL, references `riders.id`
     - `pickNumber`: integer, NOT NULL
     - `round`: integer, NOT NULL
     - `gender`: text typed as `$type<'M' | 'F'>()`, NOT NULL
     - `wasAutomatic`: boolean, NOT NULL, default false
     - `pickedAt`: timestamp with timezone, NOT NULL, defaultNow
     - Unique indexes:
       - `draft_picks_rider_league_unique` on `(leagueId, riderId)` — each rider picked once per league
       - `draft_picks_pick_number_unique` on `(leagueId, pickNumber)` — each pick slot filled once
     - Indexes:
       - `draft_picks_league_idx` on `leagueId`
       - `draft_picks_team_idx` on `teamId`

   - Drizzle relations:
     - `draftSessions` -> `leagues` (many-to-one via leagueId)
     - `draftSessions` -> `teams` (many-to-one via currentTeamId)
     - `draftPicks` -> `leagues` (many-to-one via leagueId)
     - `draftPicks` -> `teams` (many-to-one via teamId)
     - `draftPicks` -> `riders` (many-to-one via riderId)

   Import from `./leagues` for leagues/teams references and from `./riders` for riders reference. Use the same import style as existing schemas.

3. Update `src/db/schema/index.ts` to add: `export * from "./draft"`

4. Run migration:
   ```bash
   npm run db:generate
   npm run db:migrate
   ```

NOTE: Use `serial("id").primaryKey()` matching existing codebase pattern. Do NOT use `generatedAlwaysAsIdentity`. Use `uniqueIndex()` in the table options callback (same pattern as leagues.ts).

   - Env vars: `PUSHER_APP_ID`, `NEXT_PUBLIC_PUSHER_KEY`, `PUSHER_SECRET`, `NEXT_PUBLIC_PUSHER_CLUSTER`

2. Create `src/lib/pusher-client.ts`:
   - Import `PusherClient` from `'pusher-js'`
   - Export `pusherClient` singleton using `new PusherClient(key, { cluster, channelAuthorization: { endpoint: '/api/pusher-auth', transport: 'ajax' } })`
   - Env vars: `NEXT_PUBLIC_PUSHER_KEY`, `NEXT_PUBLIC_PUSHER_CLUSTER`
   - This file is ONLY imported in `"use client"` components

3. Create `src/lib/draft-snake-order.ts`:
   - Export type `DraftSlot = { pickNumber: number; round: number; gender: 'M' | 'F'; teamId: number; teamIndex: number }`
   - Export function `getTeamIndexForPick(pickIndex: number, teamCount: number): number`
     - `round = Math.floor(pickIndex / teamCount)`
     - `positionInRound = pickIndex % teamCount`
     - Return `round % 2 === 0 ? positionInRound : (teamCount - 1 - positionInRound)`
   - Export function `buildDraftOrder(teams: { id: number; name: string }[], menRounds: number, womenRounds: number): DraftSlot[]`
     - Build all pick slots for men's rounds first, then women's rounds
     - Men's picks: rounds 0 to menRounds-1, gender 'M'
     - Women's picks: rounds menRounds to menRounds+womenRounds-1, gender 'F'
     - Each round iterates over all teams using snake order (even rounds L-R, odd rounds R-L)
     - pickNumber increments globally across both drafts
   - Export function `getTotalPicks(teamCount: number, menRounds: number, womenRounds: number): number`
     - Returns `teamCount * (menRounds + womenRounds)`

4. Create `src/app/api/pusher-auth/route.ts`:
   - POST handler that authenticates user via `auth.api.getSession({ headers: await headers() })`
   - If no session, return 403 `{ error: 'Unauthorized' }`
   - Parse request body as URLSearchParams from `req.text()` (Pusher sends form-encoded)
   - Extract `socket_id` and `channel_name`
   - Look up user's team for the league (extract leagueId from channel name pattern `presence-draft-{leagueId}`)
   - Build presenceData: `{ user_id: session.user.id, user_info: { name: session.user.name, teamId: team?.id ?? null } }`
   - Call `pusherServer.authorizeChannel(socketId, channelName, presenceData)`
   - Return JSON response
   - Import from `@/lib/pusher-server`, `@/lib/auth`, and `@/lib/league-auth` for membership check

Use existing auth pattern from `src/lib/league-auth.ts` (`checkLeagueMembership`) to verify user belongs to the league before authorizing the presence channel.
