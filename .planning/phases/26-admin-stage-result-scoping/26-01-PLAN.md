Modify the `getRacesForResults()` function (lines 64-81) to add two correlated subquery columns:

1. `stagesTotal`: count of child rows in `races` where `parentRaceId = races.id`
2. `stagesWithResults`: count of child rows where `parentRaceId = races.id` AND at least one row exists in `race_results` for that child's id

Use the same SQL template literal pattern already in the query. The existing `hasResults` column uses:
```
CASE WHEN EXISTS(SELECT 1 FROM race_results WHERE race_results."raceId" = ${races.id}) THEN 1 ELSE 0 END
```

Add alongside it:
```sql
(SELECT COUNT(*) FROM races AS s WHERE s."parentRaceId" = races.id)::int AS stagesTotal
(SELECT COUNT(*) FROM races AS s WHERE s."parentRaceId" = races.id AND EXISTS(SELECT 1 FROM race_results rr WHERE rr."raceId" = s.id))::int AS stagesWithResults
```

Use `sql<number>` typed fragments. In the `.map()` at line 80, cast both new fields with `Number(r.stagesTotal ?? 0)` and `Number(r.stagesWithResults ?? 0)`.

Do NOT change any other functions. Do NOT change the `hasResults` logic. Do NOT change the file structure.

Note: Drizzle returns correlated subquery results as strings from Neon — the Number() cast handles this.
