**Add to `src/lib/transfer-queries.ts`:**

**resolveConflictingBids(leagueId: number, season: number):**
Per user decision #2: "Priority by standings — team with lowest total points gets priority."

Algorithm:
1. Fetch all pending bids for this league: `transferBids WHERE leagueId AND status = 'pending'`
2. Group bids by `inRiderId` (the free agent being claimed). Bids for different riders don't conflict.
3. For each group with >1 bid:
   a. Fetch league standings using `getLeagueStandings(leagueId, season)` from `scoring-queries.ts`
   b. Sort the conflicting bids by their team's `totalPoints` ASC (lowest points = highest priority)
   c. If two teams have equal points, use `submittedAt` ASC as tiebreaker (earlier bid wins)
   d. The winning bid is the first in sorted order
   e. All other bids for the same `inRiderId` should be marked as "rejected" with adminNote: "Outbid by waiver wire priority (lower standings)"
4. Return the ordered list of winning bids (one per unique `inRiderId`)

Return type: `Array<{ bidId: number; teamId: number; priority: number }>` — the bids in the order they should be approved.

**generateTransferWindows(leagueId: number, season: number):**
Per user decision #4: "Transfer windows auto-generated from league settings, with admin override."

Algorithm:
1. Fetch all races for the season: `races WHERE season = {season}` — only parent races (WHERE `parentRaceId IS NULL`), ordered by `startDate` ASC
2. For each race, determine window parameters based on `raceType`:
   - `grand_tour` / `womens_grand_tour`: `maxTransfers = null` (unlimited), window opens 7 days before `startDate`, closes 1 day before `startDate`
   - `high_priority_one_day`: `maxTransfers = 4`, window opens 5 days before, closes 1 day before
   - `low_priority_one_day` / `mini_tour`: `maxTransfers = 2`, window opens 3 days before, closes 1 day before
   - `world_championship`: `maxTransfers = 4`, window opens 7 days before, closes 1 day before
   - `womens_one_day`: `maxTransfers = 2`, window opens 3 days before, closes 1 day before
3. Return an array of window objects: `{ leagueId, raceId, maxTransfers, opensAt, closesAt, description: "Transfer window for {raceName}", isAutoGenerated: true }`

These are window proposals — the admin action will insert them.

**Add to `src/app/admin/transfers/actions.ts`:**

**resolveWaiverWire(leagueId: number):**
Admin action to batch-resolve all pending bids for a league using waiver wire priority:
1. `checkAdminAuth()`
2. Fetch league to get `seasonYear` from `config`
3. Call `resolveConflictingBids(leagueId, seasonYear)` to get priority-ordered winning bids
4. For each winning bid (in priority order), call the existing `approveBid(bid.bidId)` function
   - If `approveBid` throws (e.g., rider no longer free agent because a higher-priority bid just claimed them), catch the error and auto-reject the bid with note "Rider claimed by higher-priority team"
5. Return `{ success: true, approved: N, rejected: M }`

This gives the admin a "Resolve All" button that processes all pending bids at once using priority rules.

**generateWindowsForLeague(leagueId: number):**
Admin action to auto-generate transfer windows:
1. `checkAdminAuth()`
2. Fetch league to get `seasonYear`
3. Call `generateTransferWindows(leagueId, seasonYear)` to get window proposals
4. Delete existing auto-generated windows for this league (WHERE `isAutoGenerated = true AND leagueId = {leagueId}`)
5. Insert all proposed windows into `transferWindows` table
6. `revalidatePath("/admin/transfers")`
7. Return `{ success: true, windowsCreated: N }`

**createTransferWindow(data: { leagueId, raceId?, maxTransfers?, opensAt, closesAt, description }):**
Admin manual window creation:
1. `checkAdminAuth()`
2. Zod validate input
3. Insert into `transferWindows` with `isAutoGenerated: false`
4. `revalidatePath("/admin/transfers")`
5. Return `{ success: true }`

**closeTransferWindow(windowId: number):**
Admin override to close a window early:
1. `checkAdminAuth()`
2. Update window: set `closesAt = NOW()`
3. `revalidatePath("/admin/transfers")`
4. Return `{ success: true }`
