## Tech Debt by Phase

### Phase 2: Admin Backoffice
- `raceResults` schema missing `category` and `subcategory` columns — limits result entry to finish/stage_finish only
- `uniqueRaceRider` constraint (raceId, riderId) prevents same rider in multiple categories per race
- `resultAudit` is race-specific, not a generic `auditLog` pattern
- Incomplete recalculation on corrections (single result only, no position cascade)
- `previewScoringImpact` only handles finish and stage_finish categories

### Phase 3: League Management
- `nanoid` used as transitive dependency of `better-auth` — not in `package.json` directly

### Phase 4: Live Draft System
- Draft completion does not auto-transition league to "active" — owner must manually click "Start Season"
- `DraftRecap` view does not link to league detail page or prompt owner to start season

### Phase 7: Strategic Orders
- Shimanobil counter uses simplified `d.teamId !== attack.teamId` instead of rider-ownership lookup (TODO documented)
- Hammer/Innlagt Spurt/Lagtempo use admin-entered `bonusPoints` column (no auto-calculation)

### Phase 8: Race Assignment
- `getUpcomingRacesForLineup` not scoped to `league_races` (intentionally excluded from Phase 8 scope)

**Total:** 11 items across 5 phases
