<objective>
Restore lost fantasy points for riders who were hard-deleted from draftPicks (specifically
Tejada and Romeo), and prevent this from happening again for future drops.

Purpose: dropRider() currently hard-deletes the draftPicks row. All scoring queries join
through draftPicks, so deleted rows cause all historical points for that rider to vanish
from the team's total. This is a scoring data integrity bug.

Output:
- droppedAt column on draftPicks (soft delete)
- DB migration applying the column + new partial unique index
- dropRider() issues an UPDATE instead of DELETE
- All 5 scoring queries respect droppedAt: dropped riders score for races before drop date
- Tejada and Romeo re-inserted with correct droppedAt timestamps
</objective>
