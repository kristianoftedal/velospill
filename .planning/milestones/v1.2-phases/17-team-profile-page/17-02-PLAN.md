```typescript
interface PageProps {
  params: Promise<{ leagueId: string; teamId: string }>
}

export default async function TeamProfilePage({ params }: PageProps) {
  const { leagueId: leagueIdStr, teamId: teamIdStr } = await params
  const leagueId = parseInt(leagueIdStr, 10)
  const teamId = parseInt(teamIdStr, 10)

  if (isNaN(leagueId) || isNaN(teamId)) {
    notFound()
  }
  ...
}
```

Steps:
1. Parse leagueId and teamId from params — call notFound() if either is NaN
2. Call getLeagueDetails(leagueId) inside try/catch — if throws (not a member), show "League not found" message with link back to /leagues

4. Call getTeamSeasonProfile(teamId, leagueId, season) — if returns null, call notFound()
5. Render breadcrumb: Leagues › {league.name} › {profile.team.name}

The component renders three sections:

**Section 1 — Team header**
- Team name as text-4xl font-bold
- League name as muted subtitle
- Total season points displayed prominently (text-4xl, primary color), labeled "season pts"

**Section 2 — Roster summary stats**
- 3-column stats grid (matching rider profile style):
  - "Riders" (total rider count) — blue gradient card
  - "Total Points" (sum of all riders' points) — green gradient card
  - "Races Scored" (count of unique raceIds across all riders) — purple gradient card

**Section 3 — Rider roster with per-race breakdown**
- Use shadcn/ui Accordion (type="single" collapsible)
- One AccordionItem per rider, sorted by totalPoints DESC (already sorted by query)
- AccordionTrigger shows: rider name (font-semibold), pro team name (muted text-xs), gender badge, total points (bold, right-aligned)
- AccordionContent shows per-race table or list:
  - One row per race: race name, date (format "MMM d, yyyy"), points for that race
  - Each race row is itself expandable OR just flat — use flat rows for simplicity (no nested accordion needed)
  - Show category breakdown inline: for each race, show category label + position + points as small badges below the race name

Rider name in the AccordionTrigger should be a Link to /riders/{riderId} (stopPropagation to prevent accordion toggle — same pattern as Phase 16 rider list).

If a rider has isBonus: true, add a small "Bonus" Badge next to their name.
