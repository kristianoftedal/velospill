**order-queries.ts** — Create `src/lib/order-queries.ts` with these functions:

1. `getUpcomingRacesForLeague(leagueId: number, season: number)` — Returns races where `startDate > now()` for the given season. Include both parent races and stages. Return: `{ id, name, raceType, startDate, parentRaceId, stageNumber }[]`. For stages, also fetch the parent race name so the UI can display "Tour de France - Stage 3". Use LEFT JOIN on self-referencing `races.parentRaceId`.

2. `getTeamOrders(teamId: number, leagueId: number)` — Returns all orders for a team in a league, joined with orderTypes (for displayName) and races (for name, startDate). Sorted by submittedAt DESC. Return shape: `{ id, raceId, raceName, orderTypeName, orderTypeDisplayName, status, targetRiderId, targetTeamId, targetProTeam, targetCountry, bonusPoints, submittedAt }[]`.

3. `getOrderTypesForRaceType(raceType: string)` — Returns order types where the `applicableRaceTypes` JSONB array contains the given raceType. Use `sql` template literal for the JSONB containment check: `WHERE "applicableRaceTypes" @> ${JSON.stringify([raceType])}::jsonb`. Return: `{ id, name, displayName, effect, description, applicableRaceTypes }[]`.

4. `getTeamRidersForOrders(teamId: number, leagueId: number)` — Returns the team's drafted riders (id, name, team, gender). Reuse the draftPicks JOIN pattern from scoring-queries.ts.

5. `getOpponentRiders(leagueId: number, teamId: number)` — Returns riders drafted by OTHER teams in the league. Same join pattern but `ne(draftPicks.teamId, teamId)`.

6. `getOpponentTeams(leagueId: number, teamId: number)` — Returns other teams in the league (id, name). Filter by `ne(teams.id, teamId)`.

**actions.ts** — Create `src/app/(main)/leagues/[leagueId]/orders/actions.ts` with:

1. `submitOrder` server action following the exact pattern from `transfers/actions.ts`:
   - Auth check via `getAuthenticatedUser()`
   - Zod schema validation for the input: `{ leagueId: z.number(), raceId: z.number(), orderTypeId: z.number(), targetRiderId: z.number().optional(), targetTeamId: z.number().optional(), targetProTeam: z.string().optional(), targetCountry: z.string().optional(), orderConfig: z.record(z.string()).optional() }`
   - `checkLeagueMembership` to get the team
   - League status guard (must be "active")
   - **Race deadline validation**: fetch the race, verify `races.startDate > now()`. If the race is a stage (has parentRaceId), also fetch the parent to validate race type against order type.
   - **Race type compatibility**: fetch the orderType, parse `applicableRaceTypes` JSONB, verify the race's type is in the list. If the race is a stage, use the parent race's `raceType` for this check.
   - **GT-specific restriction**: if `orderType.effect` has a `restriction` field (e.g., "giro_only", "tdf_only", "vuelta_only_no_ttt"), validate the parent race name contains the expected substring ("Giro" / "Tour de France" / "Vuelta"). Log a note if restriction enforcement is based on name matching.
   - **Target validation** based on `orderType.effect.target`:
     - `"own_rider"`: verify targetRiderId is on the user's team (exists in draftPicks for this team/league)
     - `"opponent_rider"`: verify targetRiderId is on a different team's roster
     - `"opponent_all_riders"`: verify targetTeamId belongs to a different team in the league
     - `"all_own_riders"`: no target needed (applies to all own riders)
     - `"own_rider_or_country"`: requires orderConfig.kapteinChoice; if "single_rider" needs targetRiderId on own team, if "country_all" needs targetCountry
     - `"unowned_gc_top10"` (hammer): requires targetRiderId that is NOT on any team in the league
     - `"real_team"`: requires targetProTeam string
   - **World Championship guard**: if race type is "world_championship", only allow orderTypes named "kaptein" (per requirement ORDER-09: orders do not apply to World Championships except Kaptein)
   - Insert into `orders` table with status="pending"
   - `revalidatePath` for `/leagues/${leagueId}/orders` and `/admin/orders`
   - Return `{ success: true }` or `{ success: false, error: string }`

2. `cancelOrder` server action:
   - Auth + membership check
   - Fetch order, verify teamId matches, status is "pending"
   - Delete the order row (since it's still pending, deletion is cleaner than status change)
   - revalidatePath
   - Return `{ success: true }` or `{ success: false, error: string }`
