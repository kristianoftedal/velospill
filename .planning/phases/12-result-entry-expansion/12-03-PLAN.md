TTT (Team Time Trial) results are entered by team, not by individual rider. The admin selects which pro cycling teams finished 1st, 2nd, 3rd. The system then awards the TTT points to ALL riders from those teams who are drafted in any fantasy league.

**In `src/app/admin/results/actions.ts`:**

1. Add a new server action `submitTttResults`:
   ```typescript
   export async function submitTttResults(formData: {
     raceId: number
     teamPlacements: Array<{ position: number; teamName: string }>
   })
   ```

2. Implementation:
   - Validate admin auth.
   - Fetch the race to get raceType. Resolve scoring raceType (TdF detection).
   - Fetch the scoring config for category "ttt" and the resolved raceType.
   - For each team placement, calculate the points using `calculatePoints(position, scoringRules)`.
   - For each team placement, find ALL riders in the `riders` table where `riders.team` matches the `teamName`. NOTE: riders.team stores the pro team name (e.g., "UAE Team Emirates").
   - Insert a raceResult row for EACH rider on that team, with:
     - `category: "ttt"`
     - `position`: the team's placement position
     - `points`: the TTT points for that position
     - `riderId`: each rider's ID
   - Use a transaction for atomicity. Handle unique constraint violations gracefully (team might have already been entered).
   - Insert audit entry with changeType "BATCH_INSERT" and newData containing the team placements.

3. Add a `previewTttResults` action:
   ```typescript
   export async function previewTttResults(
     raceId: number,
     teamPlacements: Array<{ position: number; teamName: string }>
   )
   ```
   - Fetch scoring config for "ttt" category.
   - For each team, look up riders count and calculate: `{teamName, position, pointsPerRider, riderCount}`.
   - Return preview data showing points per team and how many riders would be affected.

4. Add a `getTeamNames` action that returns distinct team names from the riders table:
   ```typescript
   export async function getTeamNames(gender: "M" | "F"): Promise<string[]>
   ```
   Query `SELECT DISTINCT team FROM riders WHERE gender = $1 ORDER BY team`.

**In `src/components/admin/result-entry-form.tsx`:**

1. When `category === "ttt"`, render a DIFFERENT form layout:
   - Instead of rider selectors, show team name selectors (combobox with team names).
   - Each row: Position (number) + Team Name (combobox from getTeamNames).
   - Show a "Preview TTT Scoring" button that calls previewTttResults.
   - Show a "Submit TTT Results" button that calls submitTttResults.

2. Add a `teams` prop (optional, only needed for TTT):
   ```typescript
   type Props = {
     raceId: number
     riders: Rider[]
     raceType: string
     category: string
     teams?: string[]  // NEW: distinct team names for TTT
     onSuccess: () => void
   }
   ```

3. TTT form validation:
   - At least 1 team placement required.
   - Positions must be unique.
   - Team names must be unique (can't enter same team twice).

4. TTT preview display: show a simple table with Team Name, Position, Points per rider, and number of riders on that team (informational).

5. The regular rider-based form should be hidden when category is "ttt". Use a conditional:
   ```typescript
   if (category === "ttt") {
     return <TttEntrySection ... />
   }
   // existing rider-based form
   ```

   You can define `TttEntrySection` as a separate sub-component within the same file, or use a conditional block.
