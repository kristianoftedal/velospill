Create `src/lib/scoring/types.ts`:
```typescript
export type RiderScore = {
  riderId: number
  riderName: string

  subcategory: string | null
  points: number
  configDescription: string  // from scoringConfig.description for preview display

export type ScoringPreview = {
  raceId: number
  raceName: string
  raceType: string
  totalPoints: number
  results: RiderScore[]
}
```

Create `src/lib/scoring/calculate.ts`:
- Export `calculateScoresForRace(tx: Transaction | typeof db, raceId: number, results: Array<{riderId: number, position: number, category: string, subcategory?: string | null}>): Promise<ScoringPreview>`
- The function must:
  1. Fetch the race from `races` table to get raceType and name
  2. For one-day races (high/low priority, womens_one_day, world_championship): category is "finish", look up scoringConfig where raceType matches and category is "finish"
  3. For stage races (grand_tour, mini_tour, womens_grand_tour): category maps to scoringConfig category. Map incoming categories:
     - "stage_finish" -> scoringConfig category "stage_finish"
     - "sprint" -> "sprint" (or "sprint_double" based on subcategory)
     - "mountain" -> subcategory determines config category (e.g. "mountain_hc", "mountain_1cat", etc.)
     - "jersey" -> subcategory determines config category (e.g. "jersey_gc", "jersey_points", etc.)
     - "ttt" -> "ttt"
     - "end_classification" -> subcategory determines config category (e.g. "end_gc", "end_points", etc.)
  4. For each result entry, look up the matching scoringConfig row by raceType + resolved category
  5. The scoringConfig.rules JSONB has position-to-points mapping (e.g. {"1": 12, "2": 10, ...}). Look up position as string key. If position not in rules, points = 0.
  6. Handle TdF stage bonus: if race name contains "Tour de France" (or "TdF") and category is "stage_finish", also add points from "tdf_stage_bonus" config
  7. Return ScoringPreview with all calculated points
  8. Accept a Drizzle transaction OR the raw db object as first arg (for use in preview transactions and commit contexts)

Use `eq` and `and` from drizzle-orm for queries. Query scoringConfig with `validFrom <= now` and `(validUntil IS NULL OR validUntil > now)` to get current config.

Import types from drizzle: `import { db } from "@/lib/db"` for type reference. The function should accept `typeof db` or a transaction.
