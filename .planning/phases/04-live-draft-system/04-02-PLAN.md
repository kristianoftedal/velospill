1. Create `src/lib/draft-queries.ts` with shared query helpers:

   - `getAvailableRiders(leagueId: number, gender: 'M' | 'F', search?: string, filterTeam?: string, filterNationality?: string)`:
     - Subquery: get all riderIds already picked for this league+gender from `draftPicks`
     - Main query: select from `riders` where gender matches, riderId NOT IN picked subquery
     - Apply optional filters: `ilike(riders.name, '%search%')`, `eq(riders.team, filterTeam)`, `eq(riders.nationality, filterNationality)`
     - Order by `riders.name` ascending
     - Return full rider rows

   - `getDraftState(leagueId: number)`:
     - Fetch the `draftSession` for this league
     - Fetch all `draftPicks` for this league ordered by `pickNumber`
     - Fetch all teams for this league ordered by `createdAt`
     - Return `{ session, picks, teams }` or null if no session

   - `getBestAvailableRider(leagueId: number, gender: 'M' | 'F')`:
     - Call `getAvailableRiders(leagueId, gender)` with no filters
     - Return the first rider (alphabetical by name = "best available" for MVP)
     - If no riders available, return null

2. Create `src/app/(main)/leagues/[leagueId]/draft/actions.ts` with `"use server"` directive:

   **`startDraft(leagueId: number)`:**
   - Authenticate user via `auth.api.getSession`
   - Verify ownership via `checkLeagueOwnership`
   - Verify league status is `'drafting'` (fetch from leagues table)
   - Fetch all teams for the league, ordered by `createdAt`
   - If fewer than 2 teams, return error
   - Build draft order using `buildDraftOrder(teams, 18, 6)` from snake order utility
   - In a DB transaction:
     - Insert `draftSession` with status `'men'`, currentPickIndex 0, currentTeamId = first team in draft order, currentGender `'M'`, timerExpiresAt = `new Date(Date.now() + 60_000)`, startedAt = now
   - After transaction commits: schedule QStash auto-pick with 65s delay, body `{ leagueId, expectedPickIndex: 0 }`
   - After transaction commits: trigger Pusher event `'draft-started'` on `presence-draft-${leagueId}` with `{ status: 'men', currentTeamId, currentPickIndex: 0, timerExpiresAt, teams, draftOrder }`
   - `revalidatePath(\`/leagues/${leagueId}/draft\`)`
   - Return `{ success: true }`
   - QStash client: `new Client({ token: process.env.QSTASH_TOKEN! })`, publish to `${process.env.NEXT_PUBLIC_APP_URL}/api/draft/auto-pick`

   **`makePick(leagueId: number, riderId: number)`:**
   - Authenticate user
   - Fetch draft session, verify status is `'men'` or `'women'` (active draft)
   - Find user's team via `checkLeagueMembership` — verify `team.id === session.currentTeamId` (it's their turn)
   - Verify rider gender matches `session.currentGender` (can't pick a woman during men's draft)
   - Verify rider not already picked in this league (query `draftPicks` for this riderId+leagueId)
   - Get all teams ordered by `createdAt`, compute next pick state using `getTeamIndexForPick`:
     - `nextPickIndex = session.currentPickIndex + 1`
     - Determine if men's draft complete: nextPickIndex >= teamCount * 18
       - If men complete and women not started: transition status to `'women'`, reset pick tracking for women (nextPickIndex stays global), set currentGender to `'F'`
       - If men complete AND nextPickIndex >= teamCount * 24: set status `'complete'`, set completedAt
     - Compute nextTeamId using `getTeamIndexForPick(nextPickIndex, teamCount)` with appropriate offset for women's picks
   - In DB transaction:
     - Insert into `draftPicks`: leagueId, teamId, riderId, pickNumber=currentPickIndex, round, gender=currentGender, wasAutomatic=false
     - Update `draftSessions`: currentPickIndex=nextPickIndex, currentTeamId=nextTeamId, currentGender, timerExpiresAt=new Date(Date.now()+60_000) (null if complete), status (if transitioning)
   - After transaction: if not complete, schedule QStash auto-pick with 65s delay, body `{ leagueId, expectedPickIndex: nextPickIndex }`
   - After transaction: trigger Pusher `'pick-made'` on `presence-draft-${leagueId}` with: `{ pick: insertedPick, nextTeamId, nextPickIndex, timerExpiresAt, status }`
   - If draft complete: trigger Pusher `'draft-complete'` event
   - `revalidatePath(\`/leagues/${leagueId}/draft\`)`
   - Return `{ success: true, pick: insertedPick }`
   - On any validation failure: return `{ success: false, error: "descriptive message" }`

   **Important implementation notes:**
   - Trigger Pusher AFTER `db.transaction()` resolves, NOT inside the transaction callback
   - The round calculation for men: `Math.floor(pickIndex / teamCount)`. For women: `Math.floor((pickIndex - menTotalPicks) / teamCount)` where menTotalPicks = teamCount * 18
   - For snake order during women's draft: use `getTeamIndexForPick(pickIndex - menTotalPicks, teamCount)` to reset the snake pattern for women's rounds
   - Use the `{ success, error }` return pattern established in existing league actions

1. Create `src/app/api/draft/auto-pick/route.ts`:

   - Import `verifySignatureAppRouter` from `@upstash/qstash/nextjs`
   - Import draft schema, pusherServer, draft queries

   - Define inner `handler` async function accepting `NextRequest`:
     - Parse JSON body: `{ leagueId: number, expectedPickIndex: number }`
     - Fetch draft session for this league
     - **Idempotency check 1:** If no session or session.status is `'complete'` or `'pending'`, return `{ skipped: true, reason: 'no active draft' }`
     - **Idempotency check 2:** If `session.currentPickIndex !== expectedPickIndex`, return `{ skipped: true, reason: 'pick already made' }` — this means the user picked before timer expired
     - **Timing check:** If `session.timerExpiresAt && new Date() < new Date(session.timerExpiresAt.getTime() - 5000)`, return `{ tooEarly: true }` — defense against premature delivery (5s grace)
     - Call `getBestAvailableRider(leagueId, session.currentGender!)` to find the auto-pick target
     - If no rider available, return error (should not happen in a well-configured draft)
     - Compute next state (same logic as makePick — determine nextPickIndex, nextTeamId, whether status transitions)
     - In DB transaction:
       - Insert `draftPick` with wasAutomatic=true
       - Update `draftSession` with new state
     - After transaction: if not complete, schedule next QStash auto-pick (65s delay) with `{ leagueId, expectedPickIndex: nextPickIndex }`
     - After transaction: trigger Pusher `'pick-made'` event with `wasAutomatic: true` flag
     - If draft complete: trigger `'draft-complete'` event
     - Return `{ success: true, riderId, riderName, wasAutomatic: true }`

   - Export: `export const POST = verifySignatureAppRouter(handler)`
     - This wraps the handler with QStash signature verification, ensuring only QStash can call this endpoint

   - Also export the Next.js route segment config to disable body parsing for signature verification:
     ```typescript
     export const runtime = 'nodejs'
     ```

   **NOTE:** The pick-advancing logic (computing next team, round, status transitions) is the same as in makePick. To avoid duplication, extract a shared helper function `computeNextDraftState` in `src/lib/draft-queries.ts` that both makePick and auto-pick use. This function takes `(currentPickIndex, teamCount, menRounds, womenRounds)` and returns `{ nextPickIndex, nextRound, nextGender, nextTeamIndex, isComplete, isMenComplete }`.

   Add `computeNextDraftState` to `src/lib/draft-queries.ts`:
   - `nextPickIndex = currentPickIndex + 1`
   - `menTotalPicks = teamCount * menRounds`
   - `totalPicks = teamCount * (menRounds + womenRounds)`
   - `isComplete = nextPickIndex >= totalPicks`
   - `isMenComplete = nextPickIndex >= menTotalPicks`
   - `nextGender = isMenComplete ? 'F' : 'M'`
   - For round: if in men's draft `Math.floor(nextPickIndex / teamCount)`, if in women's `menRounds + Math.floor((nextPickIndex - menTotalPicks) / teamCount)`
   - For team index: if in men's draft `getTeamIndexForPick(nextPickIndex, teamCount)`, if in women's `getTeamIndexForPick(nextPickIndex - menTotalPicks, teamCount)`
