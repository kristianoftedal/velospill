  <name>Task 1: Create server actions for result entry, validation, preview, and commit</name>
  <files>src/app/admin/results/[raceId]/actions.ts</files>
  <action>
Create `src/app/admin/results/[raceId]/actions.ts` following the established pattern from races/actions.ts.

**getRaceDetails(raceId: number):**
- checkAdminAuth()
- Fetch race by id from races table
- If race has parentRaceId, it's a stage - fetch parent race too for context
- Determine available result categories based on raceType:
  - One-day races: ["finish"]
  - Grand tour stages: ["stage_finish", "sprint", "mountain", "jersey", "ttt"]
  - Grand tour (parent, for end results): ["end_classification"]
  - Mini tour stages: ["stage_finish", "sprint", "mountain", "jersey", "ttt"]
  - Mini tour (parent, for end results): ["end_classification"]
- Return race info + available categories

**getEligibleRiders(raceId: number):**
- checkAdminAuth()
- For now, return ALL riders from the riders table (eligibility filtering will come in later phases when teams/rosters exist)
- Return id, name, team sorted by name

**getExistingResults(raceId: number):**
- checkAdminAuth()
- Fetch existing results for this race from raceResults table
- Join with riders to get names
- Return existing results grouped by category

**Zod validation schema (resultEntrySchema):**
- raceId: z.number()
- category: z.string() (one of the valid categories)
- subcategory: z.string().nullable().optional() (for jersey type, mountain type, end classification type)
- results: z.array(z.object({ riderId: z.number(), position: z.number().min(1) })).min(1)
- Use superRefine for context-aware validation:
  - Validate no duplicate rider IDs in the array
  - Validate no duplicate positions in the array
  - Validate positions are sequential starting from 1

**previewResults(data: ResultEntryInput):**
- checkAdminAuth()
- Validate with resultEntrySchema
- Use db.transaction with tx.rollback() pattern:
  1. Insert results into raceResults temporarily
  2. Call calculateScoresForRace(tx, raceId, results) to get ScoringPreview
  3. tx.rollback() to discard inserts
  4. Return the ScoringPreview
- CRITICAL: Wrap in try/catch - Drizzle throws on tx.rollback(), catch the rollback error and return preview

**commitResults(data: ResultEntryInput):**
- checkAdminAuth()
- Validate with resultEntrySchema
- Use db.transaction:
  1. Calculate points using calculateScoresForRace
  2. Insert results into raceResults with calculated points
  3. Create auditLog entry for each result (operation: "CREATE")
  4. Do NOT rollback - let transaction commit
- revalidatePath("/admin/results")
- Return { success: true, preview: ScoringPreview }

Use the same checkAdminAuth pattern as existing actions.ts files. Return { success, error } pattern for error handling.
