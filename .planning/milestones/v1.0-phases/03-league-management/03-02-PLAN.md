**Create `src/app/(main)/leagues/join/[inviteCode]/actions.ts`:**

"use server" directive at top.

Import: db, leagues, teams from schema, auth, headers, revalidatePath, z, eq, and, count from drizzle-orm, isPast from date-fns, checkLeagueMembership from league-auth.

Helper `checkAuth()`: same pattern as above.

Export `validateInvite(inviteCode: string)`:
1. Query leagues where inviteCode matches, limit 1
2. If not found: return { valid: false, reason: "Invalid invite code" }
3. If inviteExpiresAt exists and isPast(inviteExpiresAt): return { valid: false, reason: "Invite link has expired" }
4. If status !== "setup": return { valid: false, reason: "League is no longer accepting new teams" }
5. Count teams where leagueId matches
6. Read maxTeams from league.config (cast config as the typed object, use .teamMax or default 10)
7. If teamCount >= maxTeams: return { valid: false, reason: "League is full" }
8. Return { valid: true, league: { id: league.id, name: league.name, ownerName: "(query user table for owner name)", teamCount, maxTeams } }
   - For the owner name: join with user table on ownerId, or do a separate query

Export `joinLeague(inviteCode: string, teamName: string)`:
1. checkAuth() to get session
2. Zod validation: teamName z.string().min(2).max(50)
3. Re-validate invite code (call the same validation logic — don't trust client state)
4. Check if user already in league: `checkLeagueMembership(session.user.id, league.id)` — if isMember, return error "You already have a team in this league"
5. Check team name uniqueness: query teams where leagueId + name match — if exists, return error "Team name already taken in this league"
6. Insert into teams: leagueId, userId: session.user.id, name: teamName
7. revalidatePath("/leagues")
8. Return { success: true, leagueId: league.id }
9. Wrap in try/catch — the unique constraints on (leagueId, userId) and (leagueId, name) provide a database-level safety net even if the application checks pass due to race conditions
