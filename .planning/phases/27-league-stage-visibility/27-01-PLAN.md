```typescript
export type StageScore = {

  raceId: number
  raceName: string
  raceType: string
  startDate: Date
  totalLeaguePoints: number  // sum of stages + endOfTour
  isMultiStage: boolean      // true for grand_tour, mini_tour, womens_grand_tour
  stages: StageScore[]       // empty for one-day races
  endOfTourPoints: number    // points from results on the parent race row itself (not a stage)
}
```

**Multi-stage race type check helper (internal constant):**
```typescript
const MULTI_STAGE_TYPES = new Set(["grand_tour", "mini_tour", "womens_grand_tour"])
```

**Refactored getLeagueRacesWithScores implementation — three queries:**

Query A — parent races with their aggregate league points (existing logic, add `isNull(races.parentRaceId)` filter to restrict to parent-only rows):
```typescript
// All parent races that have at least one result from league riders in this season
// Filter: parentRaceId IS NULL
// Returns: raceId, raceName, raceType, startDate, totalLeaguePoints
```

Query B — stage rows for multi-stage races (parentRaceId IS NOT NULL), same league/ownership-at-race-time join pattern as existing query. Returns raceId, raceName, stageNumber, startDate, parentRaceId, totalLeaguePoints. Include a `hasResults` correlated subquery:
```typescript
hasResults: sql<boolean>`EXISTS (SELECT 1 FROM race_results rr WHERE rr."raceId" = ${races.id})`
```

Query C — end-of-tour points: these are results entered against the parent race row itself. Because the parent race row has `parentRaceId IS NULL`, its results already appear in Query A's total. To isolate end-of-tour points: use the same join pattern but filter `parentRaceId IS NULL` AND the race has stages (i.e., is a multi-stage parent). The points from the parent race row itself (not its stages) represent end-of-tour classifications.

**Application-side assembly:**

1. Build parentMap from Query A rows
2. Group Query B stage rows by parentRaceId
3. For each parent: if `MULTI_STAGE_TYPES.has(raceType)`:
   - `stages` = sorted stage rows for that parent (by stageNumber ASC, then startDate ASC)
   - `endOfTourPoints` = totalLeaguePoints from Query A's parent row (since parent row points = end-of-tour results only, stages are separate rows)
   - `totalLeaguePoints` = sum(stages[i].totalLeaguePoints) + endOfTourPoints
   - `isMultiStage` = true
4. For one-day races: `stages = []`, `endOfTourPoints = 0`, `isMultiStage = false`, `totalLeaguePoints` unchanged

**Return type:** Change return signature from `Promise<LeagueRaceScore[]>` to `Promise<LeagueRaceScoreGrouped[]>`.

**Keep existing `LeagueRaceScore` type** — it is used by `StandingsHistory` and history client. Do NOT remove it. `LeagueRaceScoreGrouped` is additive.

Note: The `StandingsClient` currently imports `LeagueRaceScore` directly — plan 02 will update that import and prop type.
