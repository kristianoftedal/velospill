Per user decision #3: "Points stay with original team. Historical points scored by a rider before the transfer remain credited to the team that owned the rider at race time."

Update ALL four scoring query functions in `src/lib/scoring-queries.ts` to filter by ownership-at-race-time using `draftPicks.pickedAt` and `races.startDate`:

**getLeagueStandings:** In the LEFT JOIN chain, add a condition to the `raceResults` join (or use a WHERE clause) that only counts race results where `races.startDate >= draftPicks.pickedAt`. This ensures a transferred rider's pre-transfer race results are NOT credited to the new team. The JOIN chain is: `teams → draftPicks → raceResults → races`. Add the temporal condition on the `races` JOIN:
```
.leftJoin(
  races,
  and(
    eq(races.id, raceResults.raceId),
    eq(races.season, season),
    gte(races.startDate, draftPicks.pickedAt)  // ownership-at-race-time
  )
)
```

**getTeamRiderScores:** Same pattern. In the LEFT JOIN to `races`, add `gte(races.startDate, draftPicks.pickedAt)`.

**getRaceScoreBreakdown:** This function joins `raceResults → draftPicks → teams` using INNER JOINs. The `draftPicks` join already scopes by `leagueId`. Add a condition: `gte(races.startDate, draftPicks.pickedAt)` — but this function doesn't currently join `races`. Since it takes `raceId` as a parameter, it needs to fetch the race's `startDate` and compare against `draftPicks.pickedAt`. Two approaches:
1. Add a subquery or JOIN to `races` to get startDate, then filter `draftPicks.pickedAt <= race.startDate`
2. Since `raceId` is already known, join `races` on `raceResults.raceId` and add the temporal condition

Use approach 2: add `.innerJoin(races, eq(races.id, raceResults.raceId))` and add `lte(draftPicks.pickedAt, races.startDate)` to the draftPicks join condition.

**getLeagueRacesWithScores:** Add `gte(races.startDate, draftPicks.pickedAt)` to the draftPicks JOIN condition (which already has `eq(draftPicks.riderId, raceResults.riderId)` and `eq(draftPicks.leagueId, leagueId)`).

Import `gte` and `lte` from `drizzle-orm` (add to existing import).

IMPORTANT: For draft picks created during the original draft, `pickedAt` is the draft timestamp. Since all draft picks have `pickedAt` set to the time they were picked during the draft, and races in the season happen AFTER the draft, this condition naturally allows all original draft picks' race results through. Only transferred riders (whose new `draftPick.pickedAt` is the transfer date) will have their historical results filtered out for the new team.
