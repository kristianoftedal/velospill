  <action>
The `getRacesForResults()` function in `src/app/admin/results/actions.ts` uses a raw SQL `EXISTS` subquery for the `hasResults` field (line 73):
```ts
hasResults: sql<boolean>`EXISTS(SELECT 1 FROM ${raceResults} WHERE ${raceResults.raceId} = ${races.id})`.as('hasResults'),
```

This can fail because:
1. The `sql` template interpolation of `${raceResults}` produces the Drizzle table reference which may not resolve to the correct table name in all contexts (especially when the table name in PostgreSQL uses quotes or has casing issues with the `race_results` table).
2. The `.as('hasResults')` alias may cause issues with some Drizzle versions when used inside `.select()`.

**Fix the `getRacesForResults` function:**

Replace the `hasResults` SQL expression with a more reliable approach. Use a LEFT JOIN with a count or a subquery that explicitly references the table name:

```ts
export async function getRacesForResults() {
  await checkAdminAuth()

  const allRaces = await db
    .select({
      id: races.id,
      name: races.name,
      raceType: races.raceType,
      startDate: races.startDate,
      parentRaceId: races.parentRaceId,
      stageNumber: races.stageNumber,
      hasResults: sql<boolean>`EXISTS(SELECT 1 FROM race_results WHERE race_results."raceId" = ${races.id})`,
    })
    .from(races)
    .orderBy(desc(races.startDate))

  return allRaces
}
```

Key changes:
- Use literal table name `race_results` and column `"raceId"` (with correct casing) instead of Drizzle template interpolation `${raceResults}` and `${raceResults.raceId}` which may produce unexpected SQL.
- Remove the `.as('hasResults')` call — the field alias is already provided by the object key `hasResults` in the `.select()` call.

If the table is actually named `raceResults` (check the schema definition in `src/db/schema/results.ts` for the actual table name string), use that exact name. The schema file at `src/db/schema/results.ts` defines:
```ts
export const raceResults = pgTable("race_results", { ... })
```
So the PostgreSQL table name is `race_results` with column `raceId` (needs quotes due to camelCase).
