Apply the DDL to create the `league_races` table in Neon. Use `Pool.connect()` pattern per project convention (decision #29 in STATE.md — neon serverless sql.unsafe does not reliably commit DDL).

Write and execute a migration script inline (same approach used in phases 06 and 07):

**Step 1 — DDL:**
```sql
CREATE TABLE IF NOT EXISTS league_races (
  id SERIAL PRIMARY KEY,
  "leagueId" INTEGER NOT NULL REFERENCES leagues(id) ON DELETE CASCADE,
  "raceId" INTEGER NOT NULL REFERENCES races(id) ON DELETE CASCADE,
  "addedAt" TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE UNIQUE INDEX IF NOT EXISTS league_races_league_race_unique ON league_races ("leagueId", "raceId");
CREATE INDEX IF NOT EXISTS league_races_league_idx ON league_races ("leagueId");
CREATE INDEX IF NOT EXISTS league_races_race_idx ON league_races ("raceId");
```

**Step 2 — Pre-populate:** After the DDL, run a data migration that:
1. Queries all leagues (all statuses — setup, drafting, active, complete)
2. For each league, reads `config->'seasonYear'` (JSONB)
3. Inserts all parent races (`parentRaceId IS NULL`) for that season into `league_races`
4. Uses `ON CONFLICT DO NOTHING` to be idempotent

SQL for pre-population:
```sql
INSERT INTO league_races ("leagueId", "raceId")
SELECT l.id, r.id
FROM leagues l
CROSS JOIN races r
WHERE r.season = (l.config->>'seasonYear')::integer
  AND r."parentRaceId" IS NULL
ON CONFLICT DO NOTHING;
```

Run both via a temporary script using `Pool` from `@neondatabase/serverless` with `Pool.connect()` → `client.query()` → `client.release()`. Use `DATABASE_URL` from `process.env`. Execute with `npx tsx script.ts` then delete the script.
  </action>
