Create `src/lib/invite-codes.ts`:
```typescript
import { nanoid } from "nanoid"

/**
 * Generate a 12-character, cryptographically secure, URL-safe invite code.
 * 68 bits of entropy — collision probability ~1% after 9 billion codes.
 */
export function generateInviteCode(): string {
  return nanoid(12)
}
```

Create `src/lib/league-auth.ts`:

Import `db` from `@/lib/db`, `teams` and `leagues` from `@/db/schema/leagues`, `eq` and `and` from `drizzle-orm`, `auth` from `@/lib/auth`, `headers` from `next/headers`.

Export these functions:

**`getAuthenticatedUser()`**:
- Get session via `auth.api.getSession({ headers: await headers() })`
- If no session, throw Error("Unauthorized")
- Return session

**`checkLeagueMembership(userId: string, leagueId: number): Promise<{ isMember: boolean; team: typeof teams.$inferSelect | null }>`**:
- Query teams table where leagueId matches AND userId matches, limit 1
- Return `{ isMember: !!team, team: team ?? null }`

**`checkLeagueOwnership(userId: string, leagueId: number): Promise<boolean>`**:
- Query leagues table where id matches AND ownerId matches, limit 1
- Return `!!league`

These are utility functions used by server actions in plans 03-02 and 03-03. They follow the pattern of the existing `checkAdminAuth` helper but are reusable across the leagues feature.
