```typescript
const myTeams = await db.select({

  teamName: teams.name,

// For each team, get league info
const results = await Promise.all(myTeams.map(async (t) => {
  const [league] = await db.select().from(leagues).where(eq(leagues.id, t.leagueId)).limit(1)
  const [{ teamCount }] = await db.select({ teamCount: count() }).from(teams).where(eq(teams.leagueId, t.leagueId))
  return {
    id: league.id,
    name: league.name,
    status: league.status,

    maxTeams: (league.config as any).teamMax || 10,

  }
}))
```

Actually, for efficiency, use a single query with join:
```typescript
const result = await db
  .select({
    leagueId: leagues.id,
    leagueName: leagues.name,
    status: leagues.status,
    config: leagues.config,
    teamName: teams.name,
    createdAt: leagues.createdAt,
  })
  .from(teams)
  .innerJoin(leagues, eq(teams.leagueId, leagues.id))
  .where(eq(teams.userId, session.user.id))
  .orderBy(desc(leagues.createdAt))
```

Then compute teamCount per league separately (or use a SQL subquery). Keep it simple — two queries is fine for this scale.

**Export `getLeagueDetails(leagueId: number)`:**
1. checkAuth() to get session
2. Check membership: `checkLeagueMembership(session.user.id, leagueId)` — if not member, throw Error("Not a member of this league")
3. Fetch league: select from leagues where id = leagueId, limit 1
4. If not found, return null
5. Fetch teams: select from teams joined with user table (to get user names) where leagueId matches, order by createdAt ASC
   ```typescript
   const teamList = await db
     .select({
       id: teams.id,
       name: teams.name,
       userId: teams.userId,
       userName: user.name,
       createdAt: teams.createdAt,
     })
     .from(teams)
     .innerJoin(user, eq(teams.userId, user.id))
     .where(eq(teams.leagueId, leagueId))
     .orderBy(teams.createdAt)
   ```
6. Determine if current user is owner: league.ownerId === session.user.id
7. Return: { league (with all fields), teams: teamList, isOwner, userTeamId (from membership check) }

**Export `transitionLeagueStatus(leagueId: number, newStatus: "setup" | "drafting" | "active" | "complete")`:**
1. checkAuth() to get session
2. Check ownership: `checkLeagueOwnership(session.user.id, leagueId)` — if not owner, return { success: false, error: "Only the league owner can change status" }
3. Fetch current league status
4. Validate transition using transition map:
   ```typescript
   const validTransitions: Record<string, string[]> = {
     setup: ["drafting"],
     drafting: ["active"],
     active: ["complete"],
     complete: [],
   }
   ```
5. If transition invalid: return { success: false, error: `Cannot transition from ${current} to ${newStatus}` }
6. Additional validation for setup -> drafting:
   - Count teams in league
   - If teamCount < (config.teamMin || 2): return { success: false, error: `Need at least ${teamMin} teams to start drafting` }
7. Update leagues: set status = newStatus, updatedAt = now()
8. revalidatePath(`/leagues/${leagueId}`)
9. revalidatePath("/leagues")
10. Return { success: true, newStatus }

Follow the established Server Action pattern: try/catch, { success, error } returns.
