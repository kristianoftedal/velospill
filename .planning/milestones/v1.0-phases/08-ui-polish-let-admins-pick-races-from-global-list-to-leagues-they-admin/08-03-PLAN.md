**Part A: Update `getUpcomingRacesForLeague` in `order-queries.ts`.**

Import `leagueRaces` from schema:
```typescript
import { leagueRaces } from "@/db/schema/leagues"
```

The function already receives `leagueId` as a parameter. Add an INNER JOIN on `leagueRaces` to filter races to only those assigned to the league. The key subtlety: `getUpcomingRacesForLeague` returns BOTH parent races AND stages. Stages have `parentRaceId` set and are NOT in `league_races` (only parent races are assigned). Therefore, the join must match on EITHER:
- `races.id = leagueRaces.raceId` (for parent races), OR
- `races.parentRaceId = leagueRaces.raceId` (for stages of assigned parent races)

Use a subquery approach instead of the OR-based join for clarity:

```typescript
// Build a set of assigned race IDs for this league
const assignedRaceIds = db
  .select({ raceId: leagueRaces.raceId })
  .from(leagueRaces)
  .where(eq(leagueRaces.leagueId, leagueId))

// In the main query's WHERE clause, add:
// For parent races: races.id is in assignedRaceIds
// For stages: races.parentRaceId is in assignedRaceIds
// Combined: sql`(${races.id} IN (${assignedRaceIds}) OR ${races.parentRaceId} IN (${assignedRaceIds}))`
```

Update the `.where()` clause to add the league-scoping filter. Use Drizzle's `inArray` with the subquery, or use `sql` template for the OR condition:

```typescript
import { inArray } from "drizzle-orm"

// Replace the current .where() with:
.where(
  and(
    eq(races.season, season),
    gt(races.startDate, sql`now()`),
    sql`(${races.id} IN (SELECT "raceId" FROM league_races WHERE "leagueId" = ${leagueId}) OR ${races.parentRaceId} IN (SELECT "raceId" FROM league_races WHERE "leagueId" = ${leagueId}))`
  )
)
```

This ensures both parent races AND their stages show up if the parent is assigned.

**Part B: Update `generateTransferWindows` in `transfer-queries.ts`.**

Import `leagueRaces` from schema:
```typescript
import { leagueRaces } from "@/db/schema/leagues"
```

The function already takes `leagueId` as a parameter and already filters to `isNull(races.parentRaceId)` (parent races only). Add an INNER JOIN on `leagueRaces`:

```typescript
const parentRaceRows = await db
  .select({
    id: races.id,
    name: races.name,
    raceType: races.raceType,
    startDate: races.startDate,
  })
  .from(races)
  .innerJoin(
    leagueRaces,
    and(
      eq(leagueRaces.raceId, races.id),
      eq(leagueRaces.leagueId, leagueId)
    )
  )
  .where(
    and(
      eq(races.season, season),
      isNull(races.parentRaceId)
    )
  )
  .orderBy(asc(races.startDate))
```

Note: The local variable was originally called `parentRaces` which shadows a module-level alias. Rename the local variable to `parentRaceRows` to avoid confusion with the `parentRaces` alias used elsewhere in the file (if any). Check if there's a conflict and adjust accordingly.

**Function 1: `getLeagueStandings`**

The current query joins: `teams -> draftPicks -> raceResults -> races`. The `races` LEFT JOIN currently filters by `eq(races.season, season)` and `gte(races.startDate, draftPicks.pickedAt)`.

Add an additional condition to the `races` LEFT JOIN to also check the race (or its parent) is assigned to the league:

```typescript
// In the races LEFT JOIN, add this condition alongside the existing ones:
sql`(${races.id} IN (SELECT "raceId" FROM league_races WHERE "leagueId" = ${leagueId}) OR ${races.parentRaceId} IN (SELECT "raceId" FROM league_races WHERE "leagueId" = ${leagueId}))`
```

The full LEFT JOIN on races becomes:
```typescript
.leftJoin(
  races,
  and(
    eq(races.id, raceResults.raceId),
    eq(races.season, season),
    gte(races.startDate, draftPicks.pickedAt),
    sql`(${races.id} IN (SELECT "raceId" FROM league_races WHERE "leagueId" = ${leagueId}) OR ${races.parentRaceId} IN (SELECT "raceId" FROM league_races WHERE "leagueId" = ${leagueId}))`
  )
)
```

IMPORTANT: This is a LEFT JOIN. Using the condition in the JOIN (not WHERE) preserves the zero-point team semantics — teams with no matching race results still appear with 0 points (decision #19 in STATE.md).
