<task type="auto">
  <name>Task 1: Create admin order server actions</name>
  <files>src/app/admin/orders/actions.ts</files>
  <action>
Create `src/app/admin/orders/actions.ts` following the pattern from `src/app/admin/transfers/actions.ts`:

1. Import the `checkAdminAuth` pattern (copy the inline helper from transfers/actions.ts — it checks session + user.role === "admin").

2. `getPendingOrders()` — fetch all orders with status="pending", joined with:
   - `teams` for team name
   - `leagues` for league name
   - `races` for race name and startDate
   - `orderTypes` for displayName and effect
   - `riders` aliased as targetRider for targetRiderId (LEFT JOIN since nullable)
   - `teams` aliased as targetTeamAlias for targetTeamId (LEFT JOIN since nullable)
   Sort by submittedAt ASC (oldest first for review).
   Return: `{ orderId, leagueId, leagueName, teamId, teamName, raceId, raceName, raceStartDate, orderTypeName, orderTypeDisplayName, orderTypeEffect, status, targetRiderId, targetRiderName, targetTeamId, targetTeamName, targetProTeam, targetCountry, orderConfig, bonusPoints, submittedAt }[]`

3. `getOrderHistory()` — same query shape but WHERE status != 'pending'. Include adminNote and resolvedAt. Sort by resolvedAt DESC. Limit 50.

4. `approveOrder(orderId: number)`:
   - `checkAdminAuth()`
   - Fetch the order, verify status === "pending"
   - Update: `status = "active"`, `resolvedAt = new Date()`, `resolvedBy = session.user.id`
   - `revalidatePath("/admin/orders")`
   - Fetch the order's leagueId and `revalidatePath(/leagues/${leagueId}/orders)`
   - Return `{ success: true }` or `{ success: false, error: string }`

5. `rejectOrder(orderId: number, adminNote: string)`:
   - `checkAdminAuth()`
   - Fetch the order, verify status === "pending"
   - Update: `status = "rejected"`, `resolvedAt = new Date()`, `resolvedBy = session.user.id`, `adminNote`
   - Same revalidation
   - Return `{ success: true }` or `{ success: false, error: string }`

6. `setBonusPoints(orderId: number, bonusPoints: number)`:
   - `checkAdminAuth()`
   - Validate bonusPoints is a non-negative integer
   - Fetch the order, verify it exists and status is "active" (only active orders should have bonus points)
   - Update: `bonusPoints` on the order row
   - revalidatePath
   - Return `{ success: true }` or `{ success: false, error: string }`

Use `alias` from `drizzle-orm/pg-core` for the self-joins on riders and teams (same pattern as transfers/actions.ts).
