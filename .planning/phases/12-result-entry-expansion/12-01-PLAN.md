<objective>
Add category support to the result entry backend so race results can represent sprint, mountain, jersey, TTT, and end-of-tour classifications -- not just finish/stage_finish.

Purpose: The current raceResults schema and scoring preview only handle finish positions. All other scoring categories (sprint, mountain, jersey, TTT, end-of-tour) exist in scoringConfig but have no way to be entered. This plan adds the backend foundation.

Output: Schema with category column, category-aware scoring preview, category-aware server actions.
</objective>

  <name>Task 1: Add category column to raceResults schema and create SQL migration</name>
  <files>src/db/schema/results.ts, scripts/migrate-results-category.sql</files>
  <action>
In `src/db/schema/results.ts`:

1. Add a `category` column to the `raceResults` table:
   ```
   category: text("category").notNull().default("finish"),
   ```
   Place it after `riderId` and before `position`.

2. Update unique constraints:
   - REMOVE the `uniqueRaceRider` constraint (`race_results_race_rider_unique` on raceId, riderId) because a rider can have results in multiple categories for the same race (e.g., stage_finish position 3, sprint position 1, jersey_gc position 1).
   - REMOVE the `uniqueRacePosition` constraint (`race_results_race_position_unique` on raceId, position) because different categories will have their own position sequences.
   - ADD a new unique constraint: `uniqueRaceRiderCategory` on (raceId, riderId, category) -- a rider can only appear once per category per race.
   - ADD a new unique constraint: `uniqueRacePositionCategory` on (raceId, position, category) -- positions are unique within a category per race.

3. Add an index on category: `categoryIdx: index("race_results_category_idx").on(table.category)`

Create `scripts/migrate-results-category.sql` with the migration SQL:
```sql
-- Add category column with default 'finish' for existing rows
ALTER TABLE race_results ADD COLUMN IF NOT EXISTS "category" text NOT NULL DEFAULT 'finish';

-- Drop old unique constraints
ALTER TABLE race_results DROP CONSTRAINT IF EXISTS "race_results_race_rider_unique";
ALTER TABLE race_results DROP CONSTRAINT IF EXISTS "race_results_race_position_unique";

-- Add new unique constraints scoped by category
ALTER TABLE race_results ADD CONSTRAINT "race_results_race_rider_category_unique" UNIQUE ("raceId", "riderId", "category");
ALTER TABLE race_results ADD CONSTRAINT "race_results_race_position_category_unique" UNIQUE ("raceId", "position", "category");

-- Add category index
CREATE INDEX IF NOT EXISTS "race_results_category_idx" ON race_results ("category");
```

Note: Since drizzle-kit migrations are broken (known tech debt DEBT-01), use direct SQL migration script like the pattern established in phase 11 (see migrate-scoring-2026.ts).
