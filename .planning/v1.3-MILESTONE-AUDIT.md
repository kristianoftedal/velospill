### Integration Gaps (Tech Debt)

**GAP-01 — `dropRider` missing `return_eligible` cleanup** _(Medium severity)_

- **File:** `src/app/(main)/leagues/[leagueId]/roster/actions.ts` line 91
- **Issue:** IR cleanup uses `inArray(irRequests.status, ["pending", "approved"])` — omits `"return_eligible"`
- **Impact:** If a player drops a `return_eligible` rider from the Manage Roster page (instead of using the IR return flow), the `ir_requests` row is left behind. `getEligibleToReturnCount` continues returning > 0 — transfer block banner stays visible, `submitTransferBid` keeps rejecting bids — even though the rider is gone. Team is effectively permanently transfer-blocked until an admin manually cleans the DB.
- **Affected requirements:** IR-09 (indirect), ROST-01 (drop succeeds but leaves inconsistent state)
- **Fix:** Add `"return_eligible"` to the `inArray` at line 91. One-line change.

**GAP-02 — `submitTransferBid` slot check inconsistent with `getActiveRosterCount`** _(Low severity)_

- **File:** `src/app/(main)/leagues/[leagueId]/transfers/actions.ts` line 150
- **Issue:** Free-agent pickup slot check uses `eq(irRequests.status, "approved")` to exclude IR riders from active count. `getActiveRosterCount` uses `inArray(["approved", "return_eligible"])`. Divergence means `return_eligible` riders are not excluded in the pickup path.
- **Impact:** In practice, masked by IR-09 — players with `return_eligible` riders cannot reach this slot check (they're blocked earlier). Latent inconsistency only matters if IR-09 guard is bypassed or removed.
- **Affected requirements:** IR-06 (slot arithmetic inconsistency)
- **Fix:** Change `eq(irRequests.status, "approved")` to `inArray(irRequests.status, ["approved", "return_eligible"])` at line 150.
