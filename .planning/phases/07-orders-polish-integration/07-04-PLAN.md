Add the following functions to `src/lib/order-queries.ts` (which already has query functions from 07-02):

**Types:**
```typescript
export type ActiveOrder = {
  orderId: number
  teamId: number
  raceId: number
  orderTypeId: number
  orderTypeName: string
  effectType: string  // from orderTypes.effect.type
  effectTarget: string // from orderTypes.effect.target
  effectValues?: Record<string, number> // multiplier values per race type
  restriction?: string
  targetRiderId: number | null
  targetTeamId: number | null
  targetProTeam: string | null
  targetCountry: string | null
  orderConfig: Record<string, string> | null
  bonusPoints: number | null
}

export type OrderAdjustment = {
  teamId: number
  riderId: number | null  // null for team-level adjustments
  raceId: number
  basePoints: number
  adjustedPoints: number
  orderTypeName: string
  description: string  // human-readable explanation
}
```

**Functions:**

1. `getActiveOrdersForRace(raceId: number, leagueId: number): Promise<ActiveOrder[]>` — Join orders + orderTypes WHERE raceId AND leagueId AND status='active'. Parse the JSONB effect field into the typed fields.

2. `resolveCounters(activeOrders: ActiveOrder[]): { effectiveOrders: ActiveOrder[], counterResults: { attackOrderId: number, counterOrderId: number, description: string }[] }`:
   - Identify "attack" orders: shimanobil (zero_points), covid (half_points), bondestreik (zero_finish_points)
   - For each attack order, check if the targeted team has a "defense" order on the same race: etappeseier (double_top10_stage) or blodpose_gt (multiplier x3)
   - If counter found:
     - Mark the attack order as "countered" (remove from effectiveOrders)
     - Create a "blowback" effect: the attack's effect is redirected back to the attacker's team
     - Record in counterResults for UI display
   - If no counter: attack order remains in effectiveOrders
   - Defense orders (etappeseier, blodpose_gt) remain in effectiveOrders even if they countered something (they still provide their own positive effect)
   - Return the filtered effectiveOrders and counterResults

3. `applyOrderEffects(baseScores: { teamId: number, riderId: number, points: number, riderNationality?: string }[], effectiveOrders: ActiveOrder[], raceType: string): OrderAdjustment[]`:
   - For each effective order, compute adjustments:
     - **multiplier** (blodpose_one_day, blodpose_gt): find the rider in baseScores matching targetRiderId and teamId, multiply points by the value. Use `effectValues[raceType]` to pick the correct multiplier.
     - **zero_points** (shimanobil): find the rider in baseScores matching targetRiderId (on the TARGETED team), set points to 0.
     - **half_points** (covid): find ALL riders in baseScores on the targetTeamId, halve their points (Math.floor).
     - **double_top10_stage** (etappeseier): find all riders on the order's teamId with position <= 10, double their points.
     - **gc_position_loss** (hammer): use bonusPoints if set (admin-entered). No auto-calculation.
     - **team_sprint_points** (innlagt_spurt): use bonusPoints if set. No auto-calculation.
     - **team_placement_points** (lagtempo): use bonusPoints if set. No auto-calculation.
     - **zero_finish_points** (bondestreik): find ALL riders on the targetTeamId, set their finish points to 0 (but keep classification points if any — for simplicity, set all to 0 in Phase 7).
     - **choice** (kaptein): if orderConfig.kapteinChoice === "single_rider", multiply targetRiderId's points by 2. If "country_all", multiply all own riders matching targetCountry nationality by 1.5 (Math.floor).
     - **gammel_venn**: this is special — the targetRiderId is NOT on the order's team. Look up this rider's base points from raceResults (they may or may not be in baseScores if unowned). Add `points * multiplier` as a bonus to the order submitter's team total. Return as an OrderAdjustment with riderId=targetRiderId.
     - **double_end_tour** (sponsorens_ritt): only applies to end-of-tour stage results. Double ALL own riders' points for that race. Since this is per-stage, just double all own riders.
   - For "blowback" orders (from counter resolution, where attack bounces back): apply the attack effect to the ATTACKER's team instead of the target.
   - Return all OrderAdjustment entries

4. `getOrderAdjustedStandings(leagueId: number, season: number): Promise<{ standings: LeagueStanding[], orderAdjustments: OrderAdjustment[] }>`:
   - Call existing `getLeagueStandings(leagueId, season)` for base standings
   - Fetch ALL active orders for the league/season: query orders joined with races WHERE leagues season AND status='active'
   - Group orders by raceId
   - For each race with orders: fetch base per-rider scores, resolve counters, apply effects
   - Aggregate adjustments per team, add to base standings totals
   - Add bonusPoints from orders (Hammer, Innlagt Spurt, Lagtempo) to team totals
   - Re-rank teams after adjustment
   - Return adjusted standings + all adjustments (for UI annotation)

Important: To get per-rider per-race base points for order effect calculation, use `getRaceScoreBreakdown(raceId, leagueId)` which returns rider-level points per team. This is the data source for applying multipliers.

For Gammel Venn (unowned rider): query `raceResults` directly for the targetRiderId's points in the race, since they won't appear in the team-scoped breakdown.
