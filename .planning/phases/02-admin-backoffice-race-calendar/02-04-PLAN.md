  <files>src/app/admin/results/[raceId]/corrections/actions.ts</files>
  <action>
Create `src/app/admin/results/[raceId]/corrections/actions.ts`:

**getResultsForCorrection(raceId: number):**
- checkAdminAuth()
- Fetch all raceResults for this raceId, joined with riders for names
- Group by category + subcategory
- Return structured data: { category, subcategory, results: [{id, riderId, riderName, position, points}] }

**correctResult(data: CorrectionInput):**
- Zod schema for input:
  - resultId: z.number()
  - riderId: z.number() (can change the rider)
  - position: z.number().min(1) (can change the position)
  - reason: z.string().min(5, "Reason must be at least 5 characters") -- REQUIRED
- checkAdminAuth() and get session for userId
- Use db.transaction:
  1. Fetch current result as "before" snapshot: `SELECT * FROM raceResults WHERE id = resultId`
  2. Update raceResults: set riderId, position, updatedAt = now()
  3. Recalculate points: call calculateScoresForRace for this race to get new point values
  4. Update points on the corrected result (and potentially other results if position changes cascade)
     - Actually: recalculate ALL results for this race+category+subcategory since position changes affect relative scoring
     - For each result in the same category+subcategory, look up new points from scoringConfig based on their position
     - Update each result's points column
  5. Fetch updated result as "after" snapshot
  6. Insert auditLog entry: entityType="race_result", entityId=resultId, operation="UPDATE", userId from session, before=JSONB snapshot, after=JSONB snapshot, reason from input
- revalidatePath(`/admin/results/${raceId}/corrections`)
- Return { success: true, recalculatedResults: [...] }

**deleteResult(resultId: number, reason: string):**
- checkAdminAuth() and get session
- Zod: resultId z.number(), reason z.string().min(5)
- Use db.transaction:
  1. Fetch result as "before" snapshot
  2. Get raceId from the result
  3. Delete from raceResults
  4. Insert auditLog: operation="DELETE", before=snapshot, after=null, reason
  5. Recalculate remaining results for same race+category+subcategory (update points since positions may need adjustment)
- revalidatePath
- Return { success: true }

**getAuditHistory(raceId: number):**
- checkAdminAuth()
- Fetch from auditLog where entityType="race_result" AND entityId IN (SELECT id FROM raceResults WHERE raceId = ?)
- Also fetch audit entries where entityId was for deleted results (use a broader query: query auditLog where the before->>'raceId' = raceId OR after->>'raceId' = raceId)
- Actually simpler approach: Store raceId in the audit log. Add it to the after/before JSONB.
- Join with user table to get admin name
- Order by createdAt DESC
- Return: { id, operation, adminName, reason, before, after, createdAt }[]

Follow the established Server Action patterns: checkAdminAuth(), Zod validation, try/catch with { success, error } returns, revalidatePath after mutations.
