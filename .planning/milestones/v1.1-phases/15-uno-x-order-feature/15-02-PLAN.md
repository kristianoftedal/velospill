  <action>
    Modify `getLeagueStandings` in `src/lib/scoring-queries.ts` to include bonus rider points:

    1. Add import for `bonusRiders` from `@/db/schema/bonus-riders`.

    2. The scoring approach: bonus riders score points like regular drafted riders but ONLY for the specific GT (parent race) they were picked for. They do NOT count against roster limits.

    3. Add a separate aggregation query for bonus rider points:
       - Query: SELECT teams.id as teamId, COALESCE(SUM(raceResults.points), 0) as bonusPoints
       - FROM bonusRiders
       - INNER JOIN raceResults ON raceResults.riderId = bonusRiders.riderId
       - INNER JOIN races ON races.id = raceResults.raceId
       - WHERE bonusRiders.leagueId = leagueId
       - AND (races.id = bonusRiders.raceId OR races.parentRaceId = bonusRiders.raceId)
         — This ensures bonus rider scores include both the parent race itself and all its stages
       - AND races.season = season
       - AND league_races scoping (same subquery pattern as main standings)
       - GROUP BY bonusRiders.teamId

    4. After computing the base `ranked` standings, merge bonus rider points:
       - Create a Map from bonusPointsRows: teamId -> bonusPoints
       - For each standing, add bonusPoints to totalPoints
       - Re-sort by totalPoints DESC
       - Re-derive ranks with tie handling

    5. Similarly modify `getTeamRiderScores` to include bonus riders:
       - Add a separate query for bonus rider scores for the team
       - Append bonus rider results to the return array with a marker (e.g., `isBonus: true`)
       - This requires extending the return type to include `isBonus?: boolean`

    6. Also update `getRaceScoreBreakdown` to include bonus riders for per-race views:
       - When fetching rider breakdown for a specific race, also check if any bonus riders scored in that race
       - Join bonusRiders where bonusRiders.raceId = raceId (or parentRaceId) AND bonusRiders.leagueId = leagueId
       - Append bonus rider scores to the breakdown results

    **Important constraints:**
    - Bonus riders do NOT appear in the lineupFilter logic (they are separate from the regular draft)
    - Bonus riders do NOT need pickedAt temporal checking (they score for the entire GT they were picked for)
    - The bonusRiders.raceId is the parent GT race ID; stages belong via races.parentRaceId
